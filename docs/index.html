LeetGPU — Kernel Walkthroughs

A hands-on tour of six CUDA kernels, written for someone who has never touched a GPU before. Every optimization is grounded in the actual hardware sitting on the desk: a GeForce GTX 1660 Super (Turing TU116) — 22 SMs, 1408 cores, 6 GB GDDR6, 336 GB/s DRAM bandwidth, 96 KB combined L1/shared per SM, 1.5 MB L2, no L3, no tensor cores.

How to read this site: start with the primer if any of these terms feel fuzzy — DRAM, L1/L2 cache, shared memory, register, SM, CTA, warp, coalescing — then walk the kernels in order. Each page has step-by-step diagrams: click Step ▸ to advance, ◂ Prev to go back. Nothing animates on its own.

Kernels, in pedagogical order

Concepts build on each other. The first two teach coalescing and shared memory. The next two teach reductions and atomic operations. The last two teach kernel fusion.

step 0 · prerequisites

GPU 101 Primer

The mental model: memory hierarchy, SM, warp, coalescing, bank conflicts. Step-by-step diagrams.
step 1 · the baseline

Vector Addition

The simplest possible kernel. Used to nail down warps, lockstep execution, and what perfect coalescing looks like.
step 2 · coalescing + shared memory

Matrix Transpose

Why the naive version wastes 32× the bandwidth, and how a 32×32 shared-memory tile (with one extra column!) fixes it.
step 3 · reductions

Softmax

Three kernels, two tree-reductions, one numerical-stability trick (subtract the max), and a global atomic-add to combine block results.
step 4 · atomics + shared histograms

Top-K

A radix-bucket histogram in shared memory plus a binary search over 4-bit digits — find the K-th value without sorting.
step 5 · naive attention

Attention

Three kernel launches and one big [M × N] scores matrix sitting in DRAM. The textbook decomposition — and the bandwidth villain.
step 6 · fused + online softmax

Flash Attention

One kernel, no [M × N] buffer, streaming K/V with a running max and denominator. The trade: re-reading K, V for a clean memory footprint.
